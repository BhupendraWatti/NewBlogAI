# CODING STANDARDS

Always follow SOLID.

Always follow Clean Architecture.

Always follow DRY.

Always follow KISS.

Never duplicate business logic.

Never duplicate validation.

Never bypass Services.

Controllers must remain thin.

Business logic belongs inside Services.

Repositories handle persistence.

Policies handle authorization.

Requests handle validation.

Events handle side effects.

Jobs handle background processing.

Use Dependency Injection.

No static helper abuse.

No God Classes.

Every feature must include

Validation

Authorization

Logging

Error handling

Database transaction if required

Unit Test

Feature Test

Follow PSR standards.

Follow Laravel latest conventions.
